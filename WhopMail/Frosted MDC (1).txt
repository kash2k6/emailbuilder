# üé® Frosted-UI Overview for Rewards & Milestones App

Our app follows the **Frosted-UI** design system ‚Äî the same system powering Whop. It ensures a consistent, accessible, sleek experience across all Whop Apps. Frosted-UI is tightly integrated with TailwindCSS v4 for maximum flexibility and performance.

---

## üì¶ What Frosted-UI Includes

Frosted-UI is **not just components** ‚Äî it provides:

- Core components (Button, Card, Heading, Flex, Grid, etc.)
- Lab components (Portal, Slot, VisuallyHidden, AccessibleIcon)
- Design tokens (colors, typography, spacing)
- Tailwind plugin (automatically maps tokens into Tailwind classes)

---

## üõ† Setting Up Frosted-UI with Tailwind

Use this system: [Frosted-UI Storybook](https://storybook.whop.com)

**Only use components and helpers shown here.**
Do **not** import from other libraries like ShadCN, Radix directly, or custom atomic systems.

---

### Tailwind Plugin Installation

Add the Frosted-UI plugin in your `tailwind.config.js`:

```ts
import { frostedThemePlugin } from "frosted-ui";

export default {
  darkMode: ["class"],
  content: [],
  theme: {
    extend: {},
  },
  plugins: [frostedThemePlugin()],
};
```

Then in your global CSS (e.g., `globals.css`):

```css
@layer tw_base, frosted_ui, tw_components_utils;
@import "frosted-ui/styles.css" layer(frosted_ui);

@layer tw_base {
  @tailwind base;
}

@layer tw_components_utils {
  @tailwind components;
  @tailwind utilities;
}
```

After this setup, Frosted-UI tokens become available as Tailwind classes.

---

### üß© Tailwind Usage Principles

- Use **Frosted-UI** components for structure (Cards, Buttons, Tables, etc.).
- Use **Tailwind classes** for:
  - Background colors
  - Gradient overlays
  - Custom layouts (grid, flex, margins)
  - Responsive tweaks
- **Minimize custom CSS.** Only add it if absolutely needed.

## üîç Key Features of Frosted-UI

### 1. Breakpoints

| Size      | Screen              | Width  |
| --------- | ------------------- | ------ |
| `initial` | Phones (portrait)   | 0px    |
| `xs`      | Phones (landscape)  | 520px  |
| `sm`      | Tablets (portrait)  | 768px  |
| `md`      | Tablets (landscape) | 1024px |
| `lg`      | Laptops             | 1280px |
| `xl`      | Desktops            | 1640px |

Usage example:

```tsx
<Heading
  size={{
    initial: "3",
    md: "5",
    xl: "7",
  }}
/>
```

---

### 2. Color System

- Based on **Radix Colors**.
- 12-step scales for **accents, grays, and semantic colors**.
- Fully supports light, dark, and alpha variations.

Set via the `<Theme>` component:

```tsx
<Theme accentColor="blue" grayColor="slate">
  {/* app content */}
</Theme>
```

Supported semantic colors:

- **Danger:** ruby, tomato, red
- **Warning:** yellow, amber
- **Success:** teal, jade, green, grass
- **Info:** blue, sky

---

### 3. Typography System

- Typography scale: 1‚Äì9 (e.g., 12px‚Äì60px)
- Weights: Light (300), Regular (400), Medium (500), Bold (700)

Default fonts:

- UI: System sans-serif stack
- Code: Monospace stack

Override via CSS variables if needed:

```css
.frosted-ui {
  --default-font-family: "Inter", sans-serif;
  --code-font-family: "Fira Code", monospace;
}
```

### Frosted-UI Icons

Icons are available via the `@frosted-ui/icons` package.

Usage example:

```tsx
import { Icon } from "@frosted-ui/icons";

<Icon name="github" />;
```

---

### 4. Accessibility Helpers

Frosted-UI includes:

- `<Portal />`: Mounts content outside of DOM tree.
- `<Slot />`: Support for custom `asChild` patterns.
- `<VisuallyHidden />`: Hide elements visually but keep them readable by screen readers.
- `<AccessibleIcon />`: For accessible inline icons.

All usage matches [Radix Accessibility Patterns](https://www.radix-ui.com/primitives/docs/utilities/visually-hidden).

---

## üß† Best Practices for This App

- Follow the ‚Äî clean, calm, responsive UI.
- Components like `<Heading>`, `<Text>`, `<Flex>`, and `<Grid>` should be used for layouts.
- Tailwind used only for:
  - Backgrounds
  - Layout positioning
  - Responsive behavior
- Frosted-UI for **all major UI blocks**.
- Minimal extra libraries.
- Stay tightly scoped to components available in the [storybook](https://storybook.whop.com).
